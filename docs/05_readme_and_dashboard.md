# README and Dashboard Requirements

## README must include

### 1. Business framing
Explain that the project is about Brent under geopolitical stress, especially Iran/Hormuz-related shocks.

### 2. Why this project matters
Show relevance for:
- commodities research
- macro strategy
- trading interviews
- market risk analysis

### 3. Project objective
State clearly that this is an event-driven scenario engine, not a generic black-box forecasting project.

### 4. Methodology
Describe:
- data discovery
- price and curve loading
- event study
- news-based geopolitical factor
- regime detection
- scenario engine

### 5. Repository structure
Document folders and main Python modules.

### 6. How to run
Include:
- environment setup
- installing dependencies
- setting LSEG credentials
- running data loaders
- running Streamlit app

### 7. Key outputs
README should eventually show placeholders or actual images for:
- Brent price chart
- spread chart
- event-study chart
- regime chart
- scenario chart

### 8. Limitations
Must include honest limitations such as:
- entitlement constraints
- instrument availability uncertainty
- scenario assumptions
- model simplifications

### 9. Interview relevance
Explain why this is stronger than a simple forecasting notebook.

## Dashboard requirements

The Streamlit app should display at minimum:
- Brent price history
- selected spreads
- event window summaries
- geopolitical score
- regime classification
- scenario outputs

## Dashboard philosophy

The app should be clean and useful, not overloaded.
It should feel like a serious internal analytics dashboard.
